## 💡 설계 및 고민한 부분

> **AI 없이 스스로 설계하며 가장 신경 쓴 핵심 로직이나 아키텍처(컴포넌트 구조, 상태 관리 등), 혹은 사용자 경험(UX) 개선 사항을 적어주세요.**

- 요구사항의 세 가지 매개변수를 어떻게 해석할지 고민했습니다. `최대 별의 수`는 렌더링할 리스트 길이로 사용하고, `현재 채워진 별의 수`는 현재 선택된 평점으로 해석했습니다.
- 처음에는 세 번째 매개변수를 평점 값으로 생각해 `useEffect` 의존성 배열로 변경을 감지해야 하는지 고민했습니다. 하지만 요구사항을 다시 확인해보니 세 번째 매개변수는 평점 값이 아니라, 평점이 변경될 때 실행되는 콜백 함수라고 판단했습니다.
- 따라서 선택된 평점은 부모 컴포넌트에서 관리하고, `StarRating` 컴포넌트는 클릭 시 `onChange(rating)`을 호출해 변경된 값을 부모에게 전달하는 제어 컴포넌트 형태로 구현했습니다.

## 📖 학습한 내용 및 어려웠던 점 (선택)

> **문제를 풀며 새롭게 알게 된 개념이나, 구현 중 막혔던 부분이 있다면 공유해 주세요.**

- 처음에는 각 `li`에 `onMouseEnter`와 `onMouseLeave`를 모두 등록했습니다.
- 하지만 별 사이에 `gap`이 있는 경우, 한 별에서 다른 별로 이동하는 순간 이전 `li`의 `onMouseLeave`가 실행되어 hover 상태가 잠시 초기화되는 문제가 있었습니다.
- 이를 해결하기 위해 각 별(`li`)에는 `onMouseEnter`만 두어 현재 hover 중인 별점을 갱신하고, 전체 영역(`ul`)에 `onMouseLeave`를 두어 StarRating 위젯을 완전히 벗어났을 때만 hover 상태를 초기화하도록 변경했습니다.
