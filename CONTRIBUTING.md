# 운영 가이드 (스터디원용)

이 문서는 스터디 운영 방식과 제출 절차를 정리한 내부 가이드입니다.

## 진행 방식

- **일시:** 매주 월요일 오후 8시 (디스코드, 1시간 내외)
- **분량:** 주 1회, 1문제

| 단계           | 시간             | 내용                                                                     |
| -------------- | ---------------- | ------------------------------------------------------------------------ |
| 1. 문제 풀이   | 월요일 15:00까지 | 각자 브랜치에 코드 작성 후 Pull Request 생성 (마감)                      |
| 2. 라이브 리뷰 | 월요일 20:00~    | 디스코드 음성 채널에서 본인의 설계 의도 설명 및 질의응답, 핵심 개념 공유 |
| 3. 머지        | 리뷰 종료 후     | 각자의 PR을 `main` 브랜치에 머지하고 해당 주차 과제 종료                 |

라이브 리뷰 중 나온 핵심 개념이나 더 생각해 볼 거리는 저장소의 **Discussions** 탭에 기록하고 토론합니다.

## 과제 진행 시 유의사항 (AI 사용 관련)

본 스터디의 가장 큰 목적은 **'스스로 생각하고 구현하는 설계 능력'**을 기르는 것입니다.

- **권장:** 번역, 단순 개념 질문, 구현 방향에 대한 힌트 등 학습 보조 용도로 AI 활용
- **지양:** "코드 짜줘" 식으로 결과물을 그대로 복사 붙여넣는 행위는 금지
- **핵심:** 라이브 리뷰 시 본인의 코드를 스스로 설명하고, 팀원들의 질문에 답변할 수 있을 만큼 구조와 개념을 숙지해야 합니다.

## 디렉토리 구조 및 네이밍 컨벤션

```
assignment_test/
├── package.json
├── tsconfig.json
├── vite.config.ts
├── index.html
│
├── 01-TodoList/
│   ├── README.md
│   ├── userA/
│   ├── userB/
│   └── ...
└── 02-useQuery/
    ├── README.md
    └── ...
```

- **주차별 디렉토리:** `[문제번호]-[문제명-파스칼케이스]` (예: `01-TodoList`, `02-useQuery`)
- **개인 작업 폴더:** 문제 폴더 내부에 본인의 이름 또는 GitHub ID로 폴더 생성 (예: `01-TodoList/gildong/`)
- **작업 브랜치명:** `feat/[문제번호]-[이름]` (예: `feat/01-TodoList-gildong`)

## 사용 및 제출 방법

### 1. 초기 세팅

```bash
git clone https://github.com/hangyedan/assignment_test.git
cd assignment_test
npm install
```

### 2. 과제 확인 및 작업 브랜치 생성

```bash
git checkout -b feat/01-TodoList-gildong
```

### 3. 코드 구현 및 로컬 테스트

```bash
npm run dev
```

> **테스트 팁:** 최상위 `src/App.tsx`에 본인이 만든 컴포넌트를 불러와 자유롭게 테스트할 수 있습니다.
> 단, 로컬 테스트용 코드이므로 PR 제출 시에는 `src/App.tsx` 수정 내역을 제외하거나 삭제해 주세요.

### 4. 제출 및 리뷰 (월요일 15:00 마감)

```bash
git add .
git commit -m "feat: 01-TodoList 과제 구현(홍길동)"
git push origin feat/01-TodoList-gildong
```

이후 GitHub에서 `main` 브랜치를 대상으로 PR을 생성하고, 라이브 리뷰 전까지 다른 팀원의 코드를 미리 살펴보고 리뷰를 남깁니다.
