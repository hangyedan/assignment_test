## 📝 과제 요약

- **과제명:** Stopwatch
- **소요 시간:** 2시간

<br />

## 💡 설계 및 고민한 부분

> **AI 없이 스스로 설계하며 가장 신경 쓴 핵심 로직이나 아키텍처(컴포넌트 구조, 상태 관리 등), 혹은 사용자 경험(UX) 개선 사항을 적어주세요.**

- 스톱워치 구현에 필요한 값: 실행 상태(`isPlay`), 화면에 표시할 경과 시간(`time`), 시작 시간(`startTimeRef`)으로 분리해 관리했습니다.
- 처음에는 시작 시간과 멈춘 시간의 차이를 계산하는 방식으로 접근했지만, 화면이 실시간으로 갱신되어야 한다는 점을 고려했습니다.
- `setInterval`을 사용해 일정 간격으로 경과 시간을 다시 계산하고 화면에 반영하도록 구현했습니다.

<br />

## 📖 학습한 내용 및 어려웠던 점 (선택)

- `Date.now()`와 `performance.now()`의 차이를 학습했습니다. 경과 시간을 측정하는 스톱워치의 특성을 고려해, 시스템 시간이 아닌 페이지 로드 이후의 경과 시간을 기준으로 하는 `performance.now()`를 사용했습니다.
- `setInterval`은 렌더링과 직접 관련 없는 side effect이므로, 컴포넌트 내부에서 직접 실행하기보다 useEffect에서 생성하고 cleanup으로 정리하는 방식으로 구현했습니다.
